安全设计 — 8 层防护

#层级措施怎么做的
1传输层文件大小限制Flask MAX_CONTENT_LENGTH = 32MB，超了直接 413
2应用层频率限制滑动窗口：上传 10次/时，问答 30次/分，触发返回 429
3认证层API Token（可选）Bearer Token 装饰器，不配就跳过，配了就校验
4头部层安全响应头CSP / X-Frame-Options DENY / X-Content-Type nosniff
5输入层文件名净化 + 长度限制Unicode 规范 → 去控制字符 → 替换危险符号 → 200字符截断
6输出层零 XSS 架构全局禁用 innerHTML，只用 textContent + DOM API + 事件委托
7日志层分级异常 + 脱敏DocumentLoaderError 子类 → 友好中文；exc_info=True 记录详情
8部署层服务绑定 + 调试控制127.0.0.1 仅本地监听；FLASK_DEBUG 环境变量控制

创新点总结

📖

PDF 三级降级 OCR

PyPDF → PyMuPDF → qwen-vl-max，自动适配文本/扫描/混合型 PDF。选云端方案而非本地 OCR，因为 Python 3.13 + Windows 下 PaddleOCR 存在 NumPy 版本冲突

🎯

Cross-Encoder Rerank + 降级通知

初检 12 → 精选 4，检索质量提升性价比最高的单点改动。API 异常时自动降级，前端黄色提示让用户可感知

🔒

线程安全 KnowledgeBase Class

threading.Lock 保护所有共享状态，in-place mutation 解决 Python import 引用陷阱（重启丢数据根因）

📡

Pre-flight 检索 + SSE 取消

检索/重排/生成三阶段独立可监控可降级。stream_id + threading.Event 实现流式取消，离开页面自动停止 token 消耗

🏗

三层分离架构

routes / middleware / KnowledgeBase Class，可独立测试，不依赖 Flask test client。自定义异常分级，不再出现 500 HTML 页面

✅

TDD 测试驱动

46 项 pytest 用例含线程安全并发测试。frozen dataclass 不可变性验证。5 轮代码审查零 CRITICAL 零 HIGH

常见问题 — 应答预案

Q: 为什么用 DashScope 而不是 OpenAI？
A: 国内可直接访问，不需要代理。API 兼容 OpenAI 格式，代码里改一行 base_url 就能切。DeepSeek-V3 中文能力实测很好，性价比高于 GPT-4。
Q: Rerank 真的有用吗？会不会太慢？
A: Cross-encoder 是业界公认的 RAG 质量提升性价比最高的单点改动。我们只对初检 12 条做 rerank，延迟增加 < 500ms，几乎无感。API 挂了会降级，不影响核心功能。
Q: 能处理多大的文件？能处理多少文件？
A: 单文件 32MB 上限。ChromaDB 嵌入式向量库单机轻松支持几十万条 chunk。实际测试 300 页 PDF 正常处理。
Q: 安全性怎么保证？
A: 8 层防护：文件大小 → 频率限制 → Token 认证 → CSP 安全头 → 文件名净化 → XSS 零注入架构 → 日志脱敏 → 127.0.0.1 本地绑定。5 轮审查全部通过。
Q: 如果不用 DashScope 了，能换其他模型吗？
A: 可以。LLM 和 Embedding 都通过 LangChain 抽象，换模型只改 config.py 两行。未来计划支持 Ollama 本地模型，完全离线运行。
Q: 为什么要做状态持久化和线程安全？
A: 实际踩坑——Python import 引用陷阱导致重启后 processed_files 为空。改为 Class 封装 + in-place mutation + threading.Lock 彻底解决，并发安全也有保障。
