问题与方案

📚

痛点

企业和个人面临海量非结构化文档（PDF/Word/PPT/Excel/扫描件），传统关键词搜索无法满足语义级深度检索需求。文档格式多样，文本提取困难。

🧠

方案

构建基于 RAG 的文档智能分析系统：自动提取多格式文档文本 → 向量化存储 → Cross-Encoder 重排序 → LLM 流式生成回答。支持 19 种文件格式，PDF 三级降级 OCR。

📤

输入

PDF(含扫描版) / Word / PPT / Excel / HTML / 图片 / MD / CSV / JSON / TXT / XML / LOG — 拖拽或点击上传

💬

输出

基于文档内容的自然语言回答，SSE 流式逐字渲染，支持取消生成，Rerank 降级可见提示，聊天记录 localStorage 持久化

↕

最快 · 适用于文字型 PDF

↕
第2步: PyMuPDF (fitz) 提取
兼容更多 PDF 变体
✓ 有内容 → 返回✗ 空 → 继续
↕
第3步: 逐页渲染 PNG → DashScope qwen-vl-max OCR
扫描版/图片型 PDF · 零本地模型依赖
✓ 有内容 → 返回✗ 空 → EmptyDocumentError 友好提示

为何选择 API OCR？Python 3.13 + Windows 环境下 PaddleOCR/EasyOCR 存在 NumPy 版本冲突，DashScope 云端方案零本地模型(~2GB)，中文识别准确率高

安全设计 — 8 层防护

层级措施实现
1. 传输文件大小限制MAX_CONTENT_LENGTH = 32MB → 413
2. 应用频率限制upload: 10次/h, chat: 30次/min → 429
3. 认证API Token (可选)Bearer Token 装饰器, 未配置跳过
4. 头部安全响应头CSP / X-Frame-Options: DENY / nosniff
5. 输入文件名净化 + 长度限制sanitize_filename() + 2000 字符上限
6. 输出零 XSS 架构禁用 innerHTML · 全量 textContent · 事件委托
7. 日志分级异常 + 脱敏DocumentLoaderError → 友好中文; exc_info=True 记录详情
8. 部署服务绑定 + 调试控制127.0.0.1 · FLASK_DEBUG 环境变量

API Key 从不硬编码 · .env gitignored · 启动时 RuntimeError 校验 · 限流字典定期清理防内存泄漏
